## 9.0 从一个问题开始

你对一张加过噪声的图做边缘检测，结果边缘没找到几个，满屏都是噪点。你以为是算法的问题，于是换了个算子，噪点还是一样多。

**问题不在算法，在于：差分运算会放大高频成分，而噪声就是高频成分。**

---

## 9.1 边缘不是"线"，是灰度的突变

### 原理

边缘在物理上是"亮度在空间上发生突变的位置"。突变越大，边缘越明显。数学上，灰度的空间突变对应的是梯度——梯度幅值大=边缘强，梯度幅值小=平坦区。

三种常见的差分算子用不同的核计算梯度：

| 算子 | 核大小 | X方向核 | 特点 |
|:---|:---|:---|:---|
| Roberts | 2×2 | `[[1,0],[0,-1]]` | 最简单，只有对角差分，无平滑 |
| Prewitt | 3×3 | `[[-1,0,1],[-1,0,1],[-1,0,1]]` | 3×3简单平滑，权重均等 |
| Sobel | 3×3 | `[[-1,0,1],[-2,0,2],[-1,0,1]]` | 加权平滑，中心行权重[1,2,1]更大 |

Sobel的加权设计是有意的：中心行离当前位置最近，给更高权重；上下行离得远，权重低。这样在计算梯度幅值时，当前像素周围的信息更被重视，边缘定位更准确。

### 核心代码

```python
import cv2
import numpy as np

# Roberts算子（手动实现）
roberts_x = np.array([[1, 0], [0, -1]], dtype=np.float32)
roberts_y = np.array([[0, 1], [-1, 0]], dtype=np.float32)
G_x = cv2.filter2D(test_gray.astype(np.float32), -1, roberts_x)
G_y = cv2.filter2D(test_gray.astype(np.float32), -1, roberts_y)
roberts = np.sqrt(G_x**2 + G_y**2).astype(np.uint8)

# Sobel算子（OpenCV内置，推荐用这个）
sobel_x = cv2.Sobel(test_gray, cv2.CV_64F, 1, 0, ksize=3)
sobel_y = cv2.Sobel(test_gray, cv2.CV_64F, 0, 1, ksize=3)
sobel = np.sqrt(sobel_x**2 + sobel_y**2).astype(np.uint8)
```

### 实验现象

运行 `demo_edge.py` 实验一，你会看到：

| 算子 | 边缘粗细 | 噪声敏感度 | 原因 |
|:---|:---|:---|:---|
| Roberts | 最细 | 最高 | 核最小，无任何平滑，噪声直接放大 |
| Prewitt | 中等 | 中等 | 3×3核有平滑，但权重均等，噪声仍有一定放大 |
| Sobel | 较粗 | 最低 | 加权平滑（[1,2,1]），噪声被有效抑制 |

Roberts在斜线边缘处经常出现断点（因为只有对角差分）；Sobel的边缘更连续。**边缘强度对比**（合成图测试）：Sobel的平均边缘强度通常高于Roberts。

## 9.2 噪声：差分的天敌

### 原理

差分运算是高通滤波——它放大高频成分。噪声就是高频成分。所以差分算子天然会放大噪声：本来是平坦区域的微弱噪声，被差分一算，变成了"梯度幅值很大的点"，被误判为边缘。

**解决方案：先平滑，再差分。** 高斯滤波在做差分之前先把高频噪声压低，这样差分计算时噪声就不会被放大那么多。

为什么高斯比均值好？因为均值对边缘附近的像素也一视同仁（等权重），会把边缘也模糊掉；高斯在边缘附近权重低，所以平滑噪声的同时更好地保留了边缘信息。

### 核心代码

```python
# 错误做法：直接对噪声图做Sobel
sobel_noisy = cv2.Sobel(noisy_img, cv2.CV_64F, 1, 0, ksize=3)
sobel_noisy = np.clip(sobel_noisy, 0, 255).astype(np.uint8)

# 正确做法：先高斯平滑，再做Sobel
gaussian = cv2.GaussianBlur(noisy_img, (5, 5), sigmaX=1.5)
sobel_smooth = cv2.Sobel(gaussian, cv2.CV_64F, 1, 0, ksize=3)
sobel_smooth = np.clip(sobel_smooth, 0, 255).astype(np.uint8)
```

### 实验现象

运行 `demo_edge.py` 实验二，你会看到：

| 方法 | F1分数 | 含义 |
|:---|:---|:---|
| 直接Sobel（无预处理） | 很低 | 噪声被放大当成边缘 |
| 均值平滑+Sobel | 中等 | 噪声被压低，但边缘也被模糊了一点 |
| 高斯平滑+Sobel | 最高 | 高斯在平滑噪声的同时更好地保留了边缘位置 |

**F1分数**是边缘检测质量的综合指标（综合了"找对了多少"和"找出来的有多准"），越高越好。用原图的边缘作为ground truth，可以量化评估不同预处理方法对边缘检测质量的影响。

## 9.3 阈值分割：从边缘到区域

### 原理

边缘检测给出的是"哪些像素在边缘上"的二值答案。阈值分割则更进一步：把整张图分成"目标"和"背景"两个区域。

三种方法：
- **固定阈值**：人工指定T，简单但不够智能
- **Otsu自动阈值**：遍历0-255所有可能的T，找到让"类间方差最大化"的T——双峰分布时效果最好
- **自适应阈值**：把图像分成很多小块，每块各算一个T——光照不均时用这个

**Otsu为什么在单峰分布上失效**：Otsu的前提是直方图有明显的两个峰（目标和背景分属两个不同的亮度群体）。单峰分布（比如纯噪声图）没有两个峰，Otsu找到的T没有物理意义，分割结果几乎是随机的。

### 核心代码

```python
import cv2
import numpy as np

# 固定阈值
_, binary_fixed = cv2.threshold(img, T=128, maxval=255, type=cv2.THRESH_BINARY)

# Otsu自动阈值（会自动找最优T）
T_otsu, binary_otsu = cv2.threshold(img, 0, 255, cv2.THRESH_BINARY + cv2.THRESH_OTSU)

# 自适应阈值（分区域各算T，光照不均时用）
binary_adaptive = cv2.adaptiveThreshold(
    img, maxValue=255,
    adaptiveMethod=cv2.ADAPTIVE_THRESH_GAUSSIAN_C,
    thresholdType=cv2.THRESH_BINARY,
    blockSize=11,  # 奇数，块越大越平滑
    C=2             # 从均值中减去的常数，越大阈值越高
)
```

### 实验现象

运行 `demo_edge.py` 实验三，你会看到：

| 场景 | 固定阈值(T=128) | Otsu | 自适应阈值 |
|:---|:---|:---|:---|
| 双峰分布（目标和背景分离） | 可用但不最优 | **最优**（自动找到谷底T） | 可用 |
| 单峰分布（无明显分割点） | 无效 | **失效**（找到的T无意义） | 可用 |
| 光照不均（左暗右亮） | 左边过分割/右边欠分割 | 失效 | **最优**（各区域独立T） |

**直方图是关键**：做阈值分割之前，先看直方图。如果直方图是双峰，Otsu大概率是对的；如果直方图是单峰，Otsu大概率会失效——这时候要看场景选自适应阈值。

## 9.4 选型决策树

```
拿到一张图，先问自己：想找什么？
│
├─ 找"哪里亮度变化剧烈"（边缘在哪里）
│   ├─ 图干净吗？
│   │   ├─ 干净：直接Sobel
│   │   └─ 有噪声：先高斯平滑(σ=1.5)再做Sobel
│   │               ⚠️ 不要用均值——会把边缘也平滑掉
│   └─ 算子选型：Roberts（极快但噪）/ Sobel（最常用）
│
└─ 找"哪些区域是目标"
    ├─ 看直方图
    │   ├─ 双峰：Otsu自动阈值
    │   ├─ 单峰：自适应阈值或分水岭
    │   └─ 光照不均：自适应阈值
    └─ 参数：blockSize越大越平滑，C越大阈值越高
```

## 9.5 检验题

**第一题**：你对一张图做了边缘检测，结果噪声和边缘混在一起分不开。你尝试把核增大到7×7，噪点少了但边缘也变粗了——为什么？正确的方向是什么？

**第二题**：你有一张X光胸片（低对比度，双峰不明显），用Otsu阈值分割效果很差。你有哪几种替代思路？

**答案提示**：第一题考的是"均值/高斯核越大，平滑越强，但同时边缘也被模糊，边缘宽度增加"。正确方向是改用高斯滤波（不是增大核）来抑制噪声同时保持边缘精度。第二题考的是"自适应阈值"、"直方图规定化（用标准图做参考）"、"局部阈值"等方向。

## 本章知识树

```
边缘检测
├── 边缘=灰度突变=高梯度
│   ├── Roberts：2×2对角差分，最快但噪点最多
│   ├── Sobel：3×3加权平滑，最常用
│   └── 差分运算=高通滤波→放大噪声→必须先平滑
│
└── 阈值分割
    ├── 固定阈值：简单，但需要人工调
    ├── Otsu：自动找T，需要双峰直方图
    │   └── ⚠️ 单峰分布时Otsu失效（找到的T无物理意义）
    └── 自适应阈值：分区域各算T，光照不均时用
        └── ⚠️ blockSize越大越平滑，但细节丢失越多
```

---

## 本章核心结论

边缘检测和阈值分割是图像分割的两条路：边缘检测找"线"，阈值分割分"区域"。拿到一张图，先判断噪声情况（差分放大噪声），再选择合适的预处理（高斯>均值），最后选检测方法——图干净直接Sobel，图有噪声先滤波。阈值分割之前先看直方图：双峰用Otsu，单峰或光照不均用自适应阈值。**Otsu失效不是算法的问题，是直方图形状不满足它的前提假设。**
