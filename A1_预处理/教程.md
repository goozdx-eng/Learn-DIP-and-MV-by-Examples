# 第1阶段：图像预处理

对应教材第3、4章 | 车牌识别流水线第一步

---

## 先问一个问题：摄像头拍到的图像，计算机看到的是什么？

你以为计算机看到了一张车牌照片。实际上它看到的是一个二维整数数组，每个元素是0到255之间的一个数，表示那个像素的亮度。灰度图就是一个二维数组，彩色图是三个叠在一起的二维数组（B、G、R三个通道）。

这意味着什么？意味着一张车牌如果拍歪了10度，数组里的数字排列就和竖直拍的完全不同——同一个"8"字，歪着放的时候，像素数组和竖直放的时候几乎没有任何相似性。分类器看到这两个数组，不会认为它们是同一个东西。

所以预处理的本质是：**把五花八门的原始图像，统一变换成计算机容易处理的标准格式**。不是为了好看，是为了让后续算法能工作。

---

## 第一关：图像太暗或太亮——点运算

### 为什么会发生？

车牌在逆光下，摄像头曝光不足，整张图偏暗，字符和背景都是深灰色，区分不开。或者强光直射，整张图偏白，同样区分不开。这是最基础也最常见的问题。

### 点运算是什么意思？

"点"指的是单个像素。点运算就是对每个像素独立做一个数学映射，用公式写就是：

```
输出像素值 = f(输入像素值)
```

邻居像素的值完全不影响当前像素的计算。这是点运算区别于滤波的关键——滤波会看周围邻域，点运算不看。

### 最简单的点运算：线性拉伸

如果一张图的像素值范围只有50到180（说明对比度不足），线性拉伸把它映射到0到255：

```
输出 = (输入 - 50) / (180 - 50) * 255
```

这就是"拉伸"的直觉含义——把原来窄的分布拉宽到全范围。

### 更智能的版本：直方图均衡化

线性拉伸有个问题——你怎么知道50和180是合适的边界？手动设置很麻烦，而且不同光照条件下参数不一样。

直方图均衡化是自动版本。它的逻辑是这样的：

想象把图像的所有像素值从小到大排成一排。理想情况下，每个亮度值出现的频率应该大致相等——这样图像对比度最大。直方图均衡化就是把原始的"不均匀分布"变换成"尽量均匀的分布"。

数学上用累积分布函数（CDF）来做这件事：

```
新像素值 = 255 × CDF(原像素值)
```

CDF是从最小值到当前值的累积概率。像素越暗（累积比例越小），映射出来的值越小；像素越亮，映射出来的值越大。这个映射保证了最终分布尽量平坦。

**一个直觉验证**：如果原图90%的像素都是深灰色（值在100~150之间），均衡化之后这90%的像素会被"撑开"分布到0~229之间，对比度自然就出来了。

### 伽马校正：非线性调亮调暗

有时候不想让全图均等变化，只想提亮暗部或压暗亮部。伽马校正用指数函数做不均匀映射：

```
输出 = 输入^γ  （像素值归一化到0~1之间做计算）
```

γ < 1 时，曲线向上弯，暗部被提亮（人眼感知更自然）；γ > 1 时，曲线向下弯，亮部被压暗。

为什么叫"伽马"？因为老式CRT显示器的物理特性使得它显示亮度和输入电压的关系就是一个指数曲线，校正这个物理偏差的过程历史上叫伽马校正，名字就这么留下来了。

## 第二关：车牌拍歪了——几何变换

### 问题的实质

一辆车从斜角驶来，摄像头角度偏，拍出来的车牌是个梯形而不是矩形。更糟的是，同一辆车不同距离拍出来的车牌大小也不一样。

字符识别要求输入图像大小和方向都统一。所以需要把任意形状的车牌图像校正成标准的矩形，并缩放到固定尺寸。

### 仿射变换：保持直线的变换

仿射变换能做平移、旋转、缩放、错切，有个关键约束：**变换前平行的线，变换后仍然平行**。

用矩阵表示是这样的：

```
[x']     [a  b  c] [x]
[y']  =  [d  e  f] [y]
                   [1]
```

这个2×3矩阵的6个参数决定了做什么变换。平移只用到c和f，旋转主要用a、b、d、e，缩放让a和e不等于1。

**如何确定这个矩阵？** 找3对对应点就够了——原图上3个点和目标位置的3个点，解方程得到矩阵6个参数。OpenCV的 `getAffineTransform` 帮你解这个方程。

### 透视变换：处理梯形的变换

仿射变换解决不了透视变形。车牌斜拍之后是梯形（靠近的一边更宽，远离的一边更窄），这打破了平行关系，仿射变换做不到这种校正。

透视变换用3×3矩阵，多出来的参数就是用来处理透视效果的：

```
[x']     [h11 h12 h13] [x]
[y']  ∝  [h21 h22 h23] [y]
[1 ]     [h31 h32 1  ] [1]
```

注意这里是"∝"（成比例），不是等号——因为结果要除以第三行，这个非线性操作正是处理透视的关键所在。

需要4对对应点才能解出这9个参数（实际上有8个自由度，第9个固定为1）。所以你要找到车牌的4个角点，告诉OpenCV它们在矫正后的图像中应该在哪里：

```python
src_pts = np.float32([[角1], [角2], [角3], [角4]])  # 原图中的4个角
dst_pts = np.float32([[0,0], [w,0], [w,h], [0,h]])  # 目标矩形的4个角
M = cv2.getPerspectiveTransform(src_pts, dst_pts)
corrected = cv2.warpPerspective(img, M, (w, h))
```

### 变换后出现"空洞"怎么办——插值

几何变换本质上是把像素从一个位置搬到另一个位置。问题是，目标位置往往不是整数坐标。比如把图像旋转5度，原来位于(10, 10)的像素，在旋转后的坐标系里落在(10.3, 9.8)这种非整数位置，附近的4个整数格子里填什么？

这就是插值问题。最近邻插值直接用最近的那个整数格子的值，速度最快但会有锯齿。双线性插值用周围4个格子按距离加权平均，效果平滑，是最常用的选择。双三次插值用周围16个格子，质量最好但速度慢。

选哪个取决于场景：实时性要求高用双线性，离线处理追求质量用双三次。

## 两个容易踩的坑

**坑1：直方图均衡化在彩色图上做出来的效果很奇怪**

彩色图有三个通道，对三个通道分别做均衡化，结果是颜色失真——因为各通道亮度被独立改变了。正确做法是先转成HSV或Lab颜色空间，只对亮度通道（V或L）做均衡化，颜色通道不动。或者用CLAHE（自适应直方图均衡化），它还能避免过度均衡导致的噪声放大。

**坑2：透视变换的4个角点必须按固定顺序**

`getPerspectiveTransform` 要求src和dst的点一一对应，顺序必须一致。习惯上是左上→右上→右下→左下（顺时针）。顺序写错，车牌图像会翻转或扭曲成奇怪的形状，而且不报错，你只能从结果上发现问题。每次写这个函数时，手动检查一遍顺序是个好习惯。

## 这一阶段在整个流水线里的地位

预处理做得好，后续每一步都会变简单。做得不好，后续用再好的算法都是在错误的基础上打补丁。就像车牌如果还是梯形进入字符分割，形态学处理会把歪斜的字符切出来，特征向量就错了，最后识别率怎么都提不上去。这是"垃圾进，垃圾出"原则的最直接体现。

---

## 延伸：CLAHE比全局均衡化好在哪里

全局直方图均衡化把整张图当成一个整体处理，如果图像里有局部区域特别亮而另一些区域特别暗（比如车牌一半在阴影里），全局的CDF映射会让亮区域过曝，暗区域细节又不够。

CLAHE（Contrast Limited Adaptive Histogram Equalization）把图像分成小块，每块单独做均衡化，但加了"限幅"——每个亮度值的直方图高度不能超过阈值，超过的部分均匀分配给其他亮度，防止噪声被过度放大。这样每个局部区域都能被正确增强，是车牌场景里直方图均衡化的首选版本。
