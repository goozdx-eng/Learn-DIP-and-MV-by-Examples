# 第4阶段：特征提取

对应教材第13章 | 车牌识别流水线第四步

---

## 字符图像有了，为什么不能直接用像素值识别？

字符切割完成，每个字符是一张20×40的灰度图。最直接的想法：把这张图压平成一个向量（800个数字），送入分类器，让分类器学会"这种排列方式对应数字8"。

这个想法有三个致命问题，必须把它们想清楚。

**问题一：维度太高，样本太少**

800维特征向量，分类器要在800维空间里找分类边界。机器学习有个著名规律：数据需求随维度指数增长（维度灾难）。800维要想学得好，理论上需要几十万个样本。但实际上车牌数据集通常每类只有几百个样本。结果必然是过拟合——分类器死记住了训练集的像素排列，换一个稍微不同的光照或字体就认不出来了。

**问题二：像素值对干扰极其敏感**

同一张车牌，早上拍和傍晚拍，像素值可能差100以上。摄像头略微移动了位置，同一个"8"在图像里的坐标偏了几个像素，整个向量就完全不同了。分类器从未见过这个"新的"向量，自然识别失败。

**问题三：大量冗余**

相邻像素的颜色往往高度相关——字符笔划内部的像素值都差不多，背景区域的像素值也都差不多。用800个数字描述的信息，实际有效的可能只有几十个数字的量。

**特征提取要做的事**：把800维的原始像素转换成一个更小的、对干扰不敏感的、信息密度高的特征向量。这个过程像是压缩，但不是无损压缩——它主动丢弃对识别没用的信息（像素值的绝对大小），保留有用的信息（边缘方向、纹理模式）。

## 第一种方法：HOG——用边缘方向描述形状

### 为什么用"方向"描述字符？

字符最本质的视觉特征是它的形状，形状由轮廓决定，轮廓是像素值发生急剧变化的地方，急剧变化可以用梯度来量化，梯度有方向和幅度。

"8"的特征是有两个圆形轮廓，上下都有圆弧。圆弧在不同位置的梯度方向不同——左侧的圆弧梯度指向左，右侧指向右，顶部指向上……这些方向的分布就是"8"区别于其他字符的关键。

HOG（方向梯度直方图）就是把这种"某个区域内各梯度方向出现的频率"编码成向量。

### HOG的计算过程，一步一步来

**第一步：计算梯度**

对图像的每个像素，用Sobel算子分别计算水平方向梯度 Gx 和垂直方向梯度 Gy，然后得到：
- 梯度幅度：`M = √(Gx² + Gy²)`，表示这个像素处变化的剧烈程度
- 梯度方向：`θ = arctan(Gy/Gx)`，表示变化的方向（通常取0~180度的无符号方向）

**第二步：分Cell，统计方向直方图**

把图像分成8×8像素的小格子（Cell）。对每个Cell里的64个像素，按照它们的梯度方向，统计9个方向区间（每20度一个区间，共180度）里各方向梯度幅度的总和。

得到一个9维的向量——这就是这个Cell的局部特征，表示"这个小格子里，哪些方向的边缘最显著"。

**第三步：组Block，做归一化**

把2×2个Cell合成一个Block（即16×16像素区域），将4个Cell的9维向量拼接成一个36维向量，然后做L2归一化：

```
v_normalized = v / (||v|| + ε)
```

这一步的目的是**消除光照变化的影响**。光照变强，梯度幅度整体变大；归一化之后，向量的"方向"（各维度的相对比例）保持不变，绝对大小不重要了。

**第四步：滑动Block，拼接所有向量**

Block以一个Cell为步长在图像上滑动，每个位置产生一个36维向量，把所有位置的向量拼接起来，得到最终的HOG特征。

对于一张20×40的字符图像：Cell数=2×5=10，Block（步长1Cell）滑动位置=1×4=4，HOG总维度=4×36=144。比原始800维小了5倍多，但形状信息完整保留。

### 理解为什么HOG对光照鲁棒但对旋转不鲁棒

光照只改变像素值的绝对大小，不改变梯度的方向。Cell内的方向直方图"形状"不变，只是各个数值等比缩放。Block归一化消掉了这个缩放。所以光照变化基本不影响HOG特征。

但如果字符旋转了30度，所有梯度方向都偏转了30度，9个方向区间里的数量就全都变了，HOG向量面目全非。这就是HOG对旋转不鲁棒的原因。对于车牌字符来说，前面的预处理已经校正了旋转，所以这不是问题。

## 第二种方法：LBP——用纹理模式描述局部结构

### 一个更简单的问题：怎样描述一个像素和它邻居的关系？

HOG看梯度，LBP（局部二值模式）看的是更简单的东西：当前像素和周围邻居相比，谁更亮？

对一个3×3邻域，中心像素和周围8个像素比较大小，比中心亮的记为1，比中心暗的记为0。8个比较结果排成一个8位二进制数，就是这个像素的LBP码。

```
邻域原始值：          比较结果（1=比中心大）：
120 130 110           1  1  0
105 [115] 125    →    0  ×  1    → 二进制 11001110 → LBP码 206
 90 100 120           0  0  1
```

整张图的每个像素都算一个LBP码，得到一张LBP码图。再把这张图分成若干区域，每个区域统计LBP码的直方图，拼接所有区域的直方图就是最终特征。

### 为什么LBP对光照几乎完全不变？

LBP的操作只比较大小关系，不涉及像素值的绝对大小。如果整张图亮度加50，中心像素加50，邻居们也各自加50，相对大小关系一点都没变，LBP码完全相同。

这比HOG的光照鲁棒性还强。HOG是通过归一化来消除光照影响，LBP是从根本上就不依赖绝对值——比较大小天然就和绝对值无关。

### 均匀LBP：降维同时保留主要信息

原始LBP有256种可能的编码（8位二进制），但绝大多数出现频率极低。观察发现，编码中"0和1之间的跳变次数"不超过2次的模式（叫做均匀模式），覆盖了所有LBP码中约90%的出现次数。

均匀模式对应的是边缘（跳变1次）、角点（跳变2次）、圆斑（全0或全1）等有意义的局部结构。非均匀模式（跳变超过2次）对应随机噪声。

均匀LBP只保留58种均匀模式（加上1种"其他"类别共59维），把原来256维的直方图降到59维，而且剔除了噪声引起的随机LBP码，特征质量反而提升了。

### LBP描述纹理，HOG描述形状——它们是互补的

LBP最初是为纹理分类设计的（哪些纹理像木纹？哪些像布纹？），而不是为形状识别设计的。但实验证明它在字符识别上效果很好，原因是字符的笔划纹理（笔划的粗细、走向、交叉模式）本身就是区分字符的有效信息。

HOG捕捉的是宏观形状（大的边缘方向分布），LBP捕捉的是微观纹理（像素级的局部模式）。两者结合通常比单独使用任何一个都好。

## 第三种方法：PCA——找数据自己"告诉你"的最重要方向

### 一个关键问题：字符图像的800维特征里，真正有区分度的方向有多少？

想象所有的"8"字符图像散布在800维空间里。这些图像之间的差异（光照、字体、噪声）只让它们在少数几个维度上真正展开，其他维度上的变化要么很小，要么和这些少数几个维度高度相关。

PCA（主成分分析）就是找出这几个"真正展开"的方向。

### PCA的几何直觉

想象数据云的形状像一个压扁的橄榄球，而不是完美的球体。PCA找的是这个橄榄球的长轴方向（数据变化最大的方向，第一主成分），然后垂直于长轴找次长轴（第二主成分），以此类推。

把原始800维数据投影到前K个主成分方向上，得到K维坐标，就是PCA特征。K远小于800，但这K个坐标包含了数据中最主要的变化信息。

### 数学操作是什么？

对数据矩阵X（N个样本 × 800维，已减去均值），计算协方差矩阵：

```
C = (1/N) × X^T × X   （800×800的矩阵）
```

对C做特征值分解，得到特征值和特征向量。特征值越大，对应方向的数据方差越大，信息量越多。按特征值从大到小排列，前K个特征向量就是前K个主成分方向。

**为什么一定要先减均值？** 如果不减均值，协方差矩阵不描述数据的"形状"，而是描述数据相对于原点的分布。主成分方向会偏向数据的均值方向而不是变化最大的方向，得出错误的结论。这是PCA最容易忘记的前置操作。

### Eigenfaces（特征脸）——这个思路用在字符识别上完全一样

1991年有人把PCA用在人脸识别上，提取的主成分可视化出来像"鬼脸"，于是叫Eigenfaces。这是经典的机器视觉里程碑工作。

对字符识别，思路完全相同。把几千个字符图像做PCA，找到最能区分不同字符的主成分方向。每个字符图像在这些方向上的坐标就是特征向量。

### 一个重要约束：测试集要用训练集的主成分

PCA从训练集学习主成分方向，测试时新样本要投影到同样的方向上。如果对测试集单独做PCA，学出来的主成分方向不同，投影结果完全没有可比性，分类器就废了。

代码层面：用训练集 `pca.fit(X_train)`，再对测试集用 `pca.transform(X_test)`，而不是 `pca.fit_transform(X_test)`。

## 这三种方法，车牌识别选哪个？

没有绝对最优的选择，取决于你的约束：

对于嵌入式部署（比如STM32F4），LBP + 59维直方图是最优选择。计算量极小，不需要矩阵分解，特征提取只需要做比较和计数。

对于精度优先的PC端方案，HOG + SVM的组合精度更高，HOG捕捉形状信息更全面，是传统视觉识别的经典搭档。

PCA作为辅助降维手段——在用了HOG或LBP提特征之后，如果特征维度还是太高，再用PCA进一步压缩。单独用PCA做特征提取效果往往不如HOG和LBP，因为PCA找的是统计意义上方差最大的方向，不一定是区分类别最有效的方向（那是LDA/Fisher线性判别分析的工作）。

## LBP特征向量本身不是特征这件事很多人没想清楚

运行LBP之后得到的是一张和原图一样大小的LBP编码图，每个像素是一个0~255的LBP码。这张图不是特征向量。

把这张图分成若干小区域，对每个区域统计LBP码的直方图（各种码出现了几次），把所有区域的直方图拼接起来，才是最终的特征向量。

如果跳过这一步，直接把LBP图压平成向量，你得到的仍然是一个高维、对位置敏感的原始表示，和直接用像素值没有本质区别。直方图的作用是**聚合统计信息，消除局部位置的敏感性**。
