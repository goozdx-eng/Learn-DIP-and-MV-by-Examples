# 第3阶段：目标分离

对应教材第11、12章 | 车牌识别流水线第三步

---

## 前两步之后，计算机还是"看不见"字符

前两阶段做完，图像已经是矩形、亮度合适、噪声减少的干净图像了。但计算机面对这张图，仍然不知道哪里是字符，哪里是背景。它看到的依然是一个完整的二维像素数组，没有任何关于"字符"的概念。

目标分离要解决的问题是：**把字符区域从背景中分离出来，每个字符单独切割成一小张图**，才能逐个送入后续的识别流程。

车牌上的具体困难：
字符和背景都是深色，颜色相近；边框也是深色，和字符连在一起；还有铆钉、螺丝孔这种小黑点，如果不处理掉，会被误认为是标点符号；7个字符挤在一起，有些字符之间几乎没有间隙。

---

## 第一步：二值化——把灰度图变成黑白图

形态学操作和连通域分析都建立在二值图（只有0和255的图）上。所以第一步是把灰度图变成二值图，即决定每个像素是"前景"（字符）还是"背景"。

### 全局阈值：最简单的方案

设定一个阈值T，像素值大于T的置为255（白），小于T的置为0（黑）。车牌字符是深色的，背景是浅色的，所以字符变成0，背景变成255。再反转一下，字符变成白色，背景变成黑色——形态学处理习惯把感兴趣的前景设为白色。

问题是T怎么设？手动设对同一辆车可能有效，换一个场景光照不同又不对了。

### Otsu算法：自动找最优阈值

Otsu的想法很精妙：什么样的阈值是好的阈值？答案是**把图像分成背景和前景两类之后，这两类各自内部的方差尽量小，两类之间的方差尽量大**。

用人话说就是：好的阈值让字符区域的像素值比较一致，背景区域的像素值也比较一致，而且字符和背景两组的均值差得很开。

Otsu算法遍历所有可能的阈值（0到255），对每个候选阈值计算类间方差，选类间方差最大的那个。这个过程完全自动，不需要人工介入。

```python
_, binary = cv2.threshold(img, 0, 255, cv2.THRESH_BINARY + cv2.THRESH_OTSU)
```

第二个参数0是占位符，实际阈值由Otsu自动计算得出。

### 自适应阈值：当光照不均匀时的解法

Otsu用整张图的像素分布来算全局阈值，但如果车牌左半边在阳光下、右半边在阴影里，一个全局阈值两边都不合适。

自适应阈值把图像分成若干小块，对每小块单独计算阈值——左边亮区域用一个阈值，右边暗区域用另一个阈值。这样每个区域都能被正确二值化。

## 第二步：形态学操作——用"刷子"改造二值图

### 形态学的基本逻辑

所有形态学操作都基于同一个工具：**结构元素（SE，也叫核）**，就是一个小的二值图案，通常是矩形或椭圆形的小方块。

操作时，把SE在二值图上逐像素滑动，每个位置做一个集合运算，输出一个新的二值图。不同的集合运算产生不同的效果。

### 腐蚀：缩小白色区域

腐蚀的规则是：只有SE完全覆盖在白色区域上的位置，输出才是白色；只要SE覆盖到一个黑色像素，输出就是黑色。

效果是什么？白色区域从四周被"腐蚀"掉一圈，变小了。黑色孔洞变大了。

车牌上的铆钉是很小的黑色圆点，被字符的白色区域包围。腐蚀之后，小的白色区域（铆钉的白色周围）会消失，字符的大白色区域变小但还在。选合适的SE大小，就可以把铆钉的效果消除。

### 膨胀：扩大白色区域

膨胀和腐蚀相反：只要SE覆盖到任何一个白色像素，输出就是白色。

效果是白色区域向外扩大一圈，黑色孔洞缩小或消失。

字符笔划如果因为噪声断裂了（本来一条连续的横线中间有个小缺口），膨胀可以把断裂处填上。

### 开运算：先腐蚀再膨胀

开运算 = 腐蚀 + 膨胀。顺序是先腐蚀后膨胀。

效果：先腐蚀消灭了小噪点，再膨胀把主要目标的大小恢复回来（腐蚀损失的面积通过膨胀补回来）。净效果是**去掉小噪点，主要目标形状基本不变**。

这就是处理车牌铆钉的标准手法——开运算，SE大小调到比铆钉大、比字符小。

### 闭运算：先膨胀再腐蚀

闭运算 = 膨胀 + 腐蚀。顺序是先膨胀后腐蚀。

效果：先膨胀填充了字符内部的小孔洞，再腐蚀把扩张的轮廓收缩回来。净效果是**填充小孔洞，主要目标轮廓基本不变**。

字符"8"中间有两个圆形空洞，如果二值化后这两个空洞恰好和外部背景连通了，连通域分析会把"8"识别成三个分离的部分。闭运算把这些孔洞填上，让"8"成为一个完整的连通区域。

### 为什么一定要先二值化才能做形态学？

形态学的集合运算（腐蚀、膨胀）建立在"是前景还是背景"的二值判断上。灰度图上每个像素有256个可能值，无法做明确的集合运算。在灰度图上强行做形态学，OpenCV会对每个邻域取最小值（腐蚀）或最大值（膨胀），结果虽然能跑但意义不明确，通常和你想要的效果不符。

## 第三步：连通域分析——找出每个独立的字符

### 什么是连通域？

在二值图里，相互连接的白色像素集合叫做一个连通域。两个像素是否"连接"取决于你用哪种连通性：4连通（上下左右相邻）或者8连通（上下左右加斜角相邻）。

完成形态学处理后，每个字符应该是一个独立的连通域，背景是另一个大的连通域，可能还有一些小的噪声连通域。

### 如何用连通域分析切割字符

OpenCV的 `connectedComponentsWithStats` 不仅找出所有连通域，还告诉你每个连通域的面积、边界框（包含这个区域的最小矩形的左上角坐标、宽、高）。

```python
num_labels, labels, stats, centroids = cv2.connectedComponentsWithStats(binary)
```

`stats` 数组的第i行对应第i个连通域（标签0是背景，从1开始是目标），每行包含：
```
[x坐标, y坐标, 宽度, 高度, 面积]
```

有了边界框和面积，就可以过滤噪声和切割字符：

**面积过滤**：面积太小的是噪声，面积太大的是车牌边框（边框是最大的连通域之一）。车牌字符的面积有个合理范围，只保留这个范围内的连通域。

**宽高比过滤**：字符的高宽比大概是2:1左右，正方形的连通域大概率不是字符。

**按x坐标排序**：车牌字符从左到右排列，取出所有字符连通域的x坐标，排序就得到正确的字符顺序。

### 标签0是背景，别忘了跳过

`connectedComponentsWithStats` 的输出里，标签0永远是背景（图像中最大的黑色连通域）。从标签1开始才是真正的目标区域。写循环时从 `range(1, num_labels)` 开始，否则会把整个背景当成一个"字符"处理。

## 当字符粘连时：分水岭算法

有时候两个相邻字符之间的间隙极小，二值化后它们粘连在一起形成一个连通域。连通域分析会把这两个字符当成一个，后续识别就废了。

分水岭算法处理这种粘连问题。它的直觉来源于地理学——把二值图想象成一片地形，白色区域是"山峰"，从山峰出发注水，不同山峰流出的水会在某处"汇合"，汇合的地方就是分割线。

实际操作步骤：
1. 对二值图做**距离变换**：每个白色像素的值变成它到最近黑色像素的距离。字符的"中心"（离边缘最远的地方）会有最大的距离值。
2. 对距离变换结果做阈值，找出"确定是前景"的区域（距离最大的那些点）。
3. 对原二值图做膨胀，确定"确定是背景"的区域。
4. 两者之间的区域是"不确定区域"。
5. 以确定前景区域为"水源"运行分水岭，在不确定区域里找出分割线。

分水岭比简单连通域复杂得多，只在字符粘连这种特殊情况下才需要用到。大多数情况下，做好形态学处理后普通连通域分析就足够了。

---

## 字符切割完之后的必要操作

每个字符图像的大小此时是不同的——不同字符（"1"和"W"）自然宽度不同，切割出来的边界框大小也不同。

送入分类器之前，必须把所有字符统一resize到相同尺寸（比如20×40像素），并做归一化（像素值缩放到0~1之间）。尺寸不一致，特征向量的维度就不一样，分类器连输入格式都不匹配，根本没法工作。

---

## 汉字字符是个特殊挑战

车牌第一位是省份汉字（粤、京、沪……），笔划数量远比数字和字母复杂，二值化后内部孔洞多，形态学参数需要单独调整。而且汉字的形态学特征和数字字母差异很大，如果用同一套分类器，识别准确率会显著低于数字字母。

实践中通常把第一位省份汉字单独处理——用专门针对汉字训练的模型，或者用模板匹配（只有34个省市，直接比对34个模板）。后6位数字和字母才用主分类器。
