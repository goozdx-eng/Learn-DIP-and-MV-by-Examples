## 6.0 从一个问题开始

你拍了一张工业零件的照片，照片里有很多随机的黑点和白点——这是椒盐噪声。你用均值滤波（3×3核）去噪，结果噪声被"糊开"了，边缘也模糊了。换中值滤波，噪声去掉了，边缘却清晰。

**为什么？**

---

## 6.1 点运算：只看自己，和邻居无关

### 原理

点运算的输出像素只取决于输入像素同一个位置的值，和周围像素无关。灰度反转（255-值）、阈值分割、对比度拉伸、直方图均衡化——都是点运算。

### 核心代码

```python
import numpy as np

# 灰度反转：255减去每个像素值（只看自己，和邻居无关）
inv_img = 255 - test_img

# 对比度拉伸：把[50,200]区间拉伸到[0,255]
low, high = 50, 200
stretched = np.clip((test_img.astype(float) - low) / (high - low) * 255, 0, 255).astype(np.uint8)

# 伽马校正：γ>1提亮暗部，γ<1压暗亮部
gamma = 2.0
gamma_corrected = np.power(test_img.astype(float) / 255.0, 1/gamma) * 255
gamma_corrected = np.clip(gamma_corrected, 0, 255).astype(np.uint8)
```

### 实验现象

运行 `demo_operation_types.py` 实验一，你会看到：

- **灰度反转**：暗部变亮、亮部变暗，但像素之间的相对空间关系完全保持
- **对比度拉伸**：[50,200]的亮度范围被铺开到[0,255]，暗处更暗、亮处更亮，对比度增强
- **伽马校正（γ=2.0）**：非线性提亮暗部——暗部的放大比例远大于亮部，适合改善阴影区域可见性

关键观察：无论哪种点运算，处理前后像素之间的相对空间位置不变。点运算改变的是每个位置的"数值"，不是像素之间的"关系"。**这就是为什么点运算可以用LUT（查表）加速**——每个输出像素只查一个表项，GPU和DSP里大量用这个技巧加速。

## 6.2 线性运算：满足叠加原理，可以用卷积加速

### 原理

**叠加原理**：如果 `T(·)` 是运算，则 `T[f+g] = T[f] + T[g]` 且 `T[k·f] = k·T[f]`（k是常数）同时成立。满足这个性质的运算叫线性运算。均值滤波、高斯滤波、锐化——都是线性运算，可以写成卷积核的形式。

均值滤波的3×3核：
```
1/9  1/9  1/9
1/9  1/9  1/9
1/9  1/9  1/9
```
高斯滤波的核权重服从高斯分布，中心权重高、边缘权重低，所以平滑边缘时边缘位置信息损失更少。

### 核心代码

```python
import cv2
import numpy as np

# 均值滤波（线性）——等权重加权
kernel = np.ones((3, 3), dtype=np.float32) / 9
mean_result = cv2.filter2D(noisy_img, -1, kernel)

# 高斯滤波（线性）——加权平滑，σ越大越模糊
gaussian_result = cv2.GaussianBlur(noisy_img, (5, 5), sigmaX=1.5)
```

### 实验现象

运行 `demo_operation_types.py` 实验三，你会看到叠加原理的数值验证：

```python
# 对信号f（中心方块）和g（角落方块）分别做均值滤波，然后相加
T_f = mean_filter(f)
T_g = mean_filter(g)
T_f_plus_T_g = T_f + T_g

# 对f+g直接做均值滤波
T_f_plus_g = mean_filter(f + g)

# 误差
error = np.mean(np.abs(T_f_plus_g - T_f_plus_T_g))
# 误差 ≈ 0（在浮点精度范围内）——满足叠加原理
```

| 运算 | T[f+g] 和 T[f]+T[g] 误差 | 含义 |
|:---|:---|:---|
| 均值滤波 | ≈0 | 满足叠加原理，**线性** |
| 中值滤波 | 非零（通常几十到上百） | 不满足叠加原理，**非线性** |

**线性运算的工程价值**：可以拆开分别处理再合并，结果不变；可以用卷积定理在频域加速（大核滤波时比空间域快得多）。

## 6.3 非线性运算：不满足叠加原理，但对椒盐噪声特别有效

### 原理

中值滤波是最典型的非线性运算。输出像素是邻域9个像素的中位数（中值），不是平均值。中位数不受极端值影响——9个数里有一个是0（黑点噪声），其余8个接近180-200，中位数会是180左右（接近正常值），但均值会把0也平均进去，结果被拉低。

**椒盐噪声为什么中值滤波有效**：椒盐是极端值（0或255），中值滤波天然"忽略"它们——排序后取中间位置，极端值落在两端，根本不会被选中。均值不行，因为它会把极端值平均进去。

### 核心代码

```python
# 中值滤波（非线性）——无法写成卷积形式，必须逐像素排序
median_result = cv2.medianBlur(noisy_img, k=3)
```

### 实验现象

运行 `demo_operation_types.py` 实验二，你会看到：

| 方法 | k=3 PSNR | k=5 PSNR | k=7 PSNR | 观察 |
|:---|:---|:---|:---|:---|
| 均值滤波 | ~低 | ~更低 | ~更低 | 核越大，噪声被"糊开"越严重，边缘也越模糊 |
| 中值滤波 | ~高 | ~更高 | ~更高 | 核越大，椒盐去除越干净，但文字边缘开始模糊 |

**PSNR越高越好**，中值滤波在所有核大小下PSNR都显著高于均值滤波。关键机制：均值把噪声点的值**稀释**到周围9个像素（每个被稀释到约1/9），但整张图变模糊；中值滤波**直接扔掉**噪声点（极端值排序后落在两端，不影响中位数取值）。

## 6.4 选型决策树

```
这张图有噪声？
│
├─ 高斯噪声（随机分布的微小扰动）
│   └─ 首选高斯滤波（加权平均，边缘附近权重小）
│      └─ 边缘检测前：先高斯平滑再差分
│
├─ 椒盐噪声（随机黑点0和白点255）
│   └─ 首选中值滤波（忽略极端值）
│      └─ ⚠️ 计算量大，嵌入式优化方向：快速选择算法
│
└─ 均匀噪声
    └─ 均值滤波（简单有效）
```

## 6.5 检验题

**第一题**：你在一张所有像素都是128的图上，只有一个像素是255（噪声点）。你用3×3均值滤波，输出图像的噪声点位置的值是多少？如果用3×3中值滤波呢？

**第二题**：你在嵌入式边缘设备上做实时去噪，椒盐噪声会周期性地出现（工厂电火花干扰）。算力有限，选均值还是中值？有没有进一步优化的空间？

**答案提示**：第一题考的是"均值把噪声点的值平均到周围9个像素（被稀释到约1/9）；中值直接扔掉噪声点（取中位数，9个数排序后中位数是第5个，不是噪声点）"。第二题考的是"中值滤波更有效但计算量大，排序9个数需要9次比较，优化方向是快速选择算法（Quickselect）"。

## 本章知识树

```
运算类型
├── 点运算：只看自己，不看邻居
│   ├── 灰度反转、阈值分割、对比度拉伸、直方图均衡化
│   └── 可用LUT查表加速，GPU/DSP常用

│   ├── 均值滤波：等权重加权

│   └── 可用卷积加速，大核用频域优化
└── 非线性运算：不满足叠加原理
    └── 中值滤波：对椒盐噪声有效（忽略极端值）

```

## 本章核心结论

四种运算类型是选择算法的底层框架。拿到一张图，先判断它有什么类型的噪声——高斯用高斯滤波，椒盐用中值滤波；先判断要做什么操作——边缘检测前要平滑但不要过度（高斯优于均值），实时系统要评估算力（查表加速点运算，卷积定理加速线性大核滤波）。知道一个运算是线性还是非线性，是判断它适用场景的第一步。
