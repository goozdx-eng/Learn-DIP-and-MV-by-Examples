# 第5阶段：分类识别

对应教材第15、16、17章 | 车牌识别流水线最后一步

## 分类器的本质：在特征空间里画边界

经过前四步，每个字符图像已经被转换成了一个N维特征向量（比如LBP特征是59维）。问题变成了：给定这个向量，判断它属于36个字符类别（0-9、A-Z）中的哪一类？

换一种说法：在59维的特征空间里，36类字符各自形成一个"点云"。分类器的工作就是在这个空间里画出36个区域的边界，让属于"8"的点落在"8"的区域里，属于"B"的点落在"B"的区域里。

不同分类器的区别在于：**怎样画这个边界，以及边界形状有什么约束**。

## SVM：找最宽的街道

### 直觉：不仅要分对，还要分得最稳

假设你要在白纸上用一条直线把两组点分开。能画出无数条分界线，都能完全分对。哪条最好？

SVM的回答是：找让两侧"空白最宽"的那条线。这条线到两侧最近的点的距离最大，叫做最大间隔。直觉上，间隔越宽，新来的点落错区域的可能性越小——分类器对测试数据的容错能力越强。

那些决定了边界位置的"最近的点"叫做**支持向量**——只有这几个点是真正决定决策边界的，其他点你拿走都不影响边界的位置。这是SVM名字的来源，也是为什么SVM在样本少的时候仍然好用的原因：它不需要所有训练数据，只依赖那几个支持向量。

### 数学约束是什么？

设决策边界为 `w·x + b = 0`，两侧的支持向量满足 `w·x + b = ±1`。间隔宽度 = 2/||w||。

最大化间隔等价于最小化 ||w||²：

```
min  (1/2)||w||²
s.t. y_i(w·x_i + b) ≥ 1   对所有训练样本
```

这是一个标准的二次规划问题，有唯一最优解。

### 真实数据不是线性可分的——核技巧

字符特征向量在59维空间里，"8"的点云和"B"的点云大概率不是线性可分的（用一个超平面无法完全分开）。

SVM用核技巧解决这个问题。核函数的想法是：不需要真正把数据映射到高维空间，只需要能计算"在那个高维空间里两个点的内积"就够了。这个计算快捷方式就是核函数 K(x, y)。

**RBF（径向基）核** `K(x,y) = exp(-γ||x-y||²)` 是字符识别中最常用的。它的几何意义是：两个点越近（欧氏距离小），内积越大（接近1）；两个点越远，内积越小（接近0）。γ控制"近"的判断标准——γ大的时候，只有非常近的点才被认为"接近"，决策边界更曲折；γ小的时候，影响范围更广，边界更平滑。

### C参数：允许多少错误？

真实数据几乎不可能完全线性可分，SVM引入了松弛变量允许部分错误，C是惩罚系数：

C 很大 → 对错误惩罚重 → 分类器努力不犯错 → 边界弯折更多 → 过拟合风险
C 很小 → 对错误宽容 → 边界更平滑 → 泛化能力更强 → 欠拟合风险

车牌识别经验值：C=10, γ=0.001，但这需要用交叉验证来调整，不是固定答案。

### SVM用于多分类

原始SVM只能做二分类（+1 或 -1）。36类字符要做多分类，OpenCV的实现用的是**One-vs-Rest**策略：训练36个二分类SVM，每个SVM把"该类 vs 其他所有类"分开。预测时，让36个SVM都投票，置信度最高的那个类就是结果。

## ANN/MLP：让神经元学会"投票"

### 神经元的基本操作

单个神经元做的事情：对输入向量做加权求和，加上偏置项，再过一个非线性函数：

```
输出 = f(w₁x₁ + w₂x₂ + ... + wₙxₙ + b)
```

f 是激活函数。如果没有激活函数，多层神经元叠加后仍然等价于一层线性变换，加再多层也没意义。非线性激活函数让网络能表达复杂的非线性关系。

**为什么Sigmoid慢慢被ReLU取代？**

Sigmoid的输出在(-∞, +∞) → (0, 1)之间。当输入很大或很小的时候，sigmoid的导数接近0（饱和区），梯度在反向传播时每经过一层都被乘以一个接近0的数，经过很多层之后梯度趋近于0，前面的层几乎学不到东西。这叫梯度消失问题。

ReLU就是 `max(0, x)`，正数区间导数恒为1，梯度可以畅通无阻地传回去。这个简单的改动让训练很深的网络成为可能。

### 反向传播的直觉

网络前向计算得到预测结果，和正确答案比较，用损失函数（比如交叉熵）量化"错了多少"。反向传播用链式法则把这个"错误信号"从输出层一路传回输入层，计算每个权重对错误的贡献（偏导数），按比例调整权重减小误差。

这个过程重复很多轮，每轮都用新的数据（或者不同批次），权重逐渐收敛到一个"错误最小"的状态。

**OpenCV内置的ANN_MLP**不需要PyTorch或TensorFlow就能用，适合嵌入式部署。对于车牌字符识别这种相对简单的问题，一个输入层 + 一个隐藏层（50~100个神经元）+ 输出层（36类）就足够了，而且推理速度很快。

## AdaBoost：弱的加在一起也可以很强

### 弱分类器是什么？

弱分类器指只比随机猜（50%）稍微好一点点的分类器。比如对图像的某一个像素做阈值判断："如果位置(15,20)的像素值大于130，就判断是字符A"——这种极其简单的规则，准确率可能只有55%，但它是一个弱分类器。

### AdaBoost的思路

一个弱分类器不够用，但如果组合几百个弱分类器，每个负责一小块，整体准确率会大幅提升。

关键在于**怎么组合**。不是简单地让所有分类器投票，AdaBoost用了一个聪明的策略：

第一轮，所有训练样本权重相等，训练一个弱分类器，记录它分错了哪些样本。

第二轮，被分错的样本权重加大（让下一个分类器重点关注这些困难样本），训练另一个弱分类器。

如此迭代T轮，每一轮的分类器都专注于解决上一轮的失败案例。

最终，T个弱分类器按各自的准确率加权投票：准确率高的分类器话语权大（权重 α 大），准确率低的话语权小：

```
最终结果 = sign(α₁h₁(x) + α₂h₂(x) + ... + αₜhₜ(x))
```

其中分类器的权重 `α = (1/2) × ln((1-错误率)/错误率)`，错误率越低 α 越大。

### AdaBoost的一个典型应用：Viola-Jones人脸检测

2001年Viola和Jones提出的实时人脸检测算法，用简单的Haar特征作为弱分类器，AdaBoost组合几百个，再用级联结构加速。这个算法在2001年能在普通PC上实时运行，轰动一时，OpenCV里的 `cv2.CascadeClassifier` 就是这个算法的实现。

### AdaBoost对噪声的脆弱性

被标错的噪声样本（本来是"8"被标成了"B"）在每轮迭代中都会被当成"困难样本"权重持续增加，最终分类器会强行"学会"这个错误样本，导致过拟合。如果训练数据里有标注错误，AdaBoost会比SVM和ANN更受影响。

## 分类器评估：不要只看准确率

**混淆矩阵**是最直观的评估工具。36×36的矩阵，行代表真实类别，列代表预测类别，对角线元素是正确分类数。

只看准确率有时候会误导你。比如"1"和"I"（大写字母i）在某些字体里几乎一样，分类器可能把大量的"I"预测成"1"，整体准确率还是95%，但"I"的召回率可能只有60%。混淆矩阵能立刻定位到"哪两个类别最容易混淆"，指导你有针对性地改进。

**查看混淆矩阵中数值最大的非对角元素**，就能找到分类器最大的弱点。针对这些容易混淆的类别对，要么收集更多该类别的训练数据，要么针对这些类别设计更有区分度的特征。

## 为什么说LBP + SVM是车牌字符识别的经典组合？

不是因为它"最好"，而是因为在传统方法框架内它在工程约束下最均衡：

LBP特征提取几乎不需要浮点运算，只做比较和计数，在没有FPU的MCU上也能快速运行。SVM训练好之后推理只是一次向量内积运算，速度极快。整套方案可以在STM32F4这样的低端MCU上以30fps运行，不需要专用AI芯片。

同时，LBP对光照变化的天然鲁棒性和SVM的最大间隔特性配合很好——特征稳定 + 分类边界宽，给了系统很强的容错空间。这就是为什么这个组合在2010年前后成为车牌识别的工业标准，即使今天深度学习已经全面超越它，在边缘计算场景里它仍然是首选。

---

## 深度学习取代了流水线，但你仍然要学这套

现在主流的车牌识别（比如LPRNet、CRNN）是端到端深度学习，不需要手动做特征提取，直接把图像塞进CNN，输出字符序列。精度比传统方法高，在有GPU的服务器上速度也更快。

但理解传统流水线有不可替代的价值：你知道"为什么图像要预处理"、"为什么不能直接用像素做分类"、"什么是好的特征"——这些问题的答案，是理解深度学习"内部在做什么"的基础。CNN的每一层本质上是在自动学习类似HOG、LBP这样的特征；Batch Normalization本质上是在做类似直方图均衡化的操作；深度网络的池化层起到了类似连通域分析的区域聚合作用。

学明白了传统方法，再看深度学习，会清楚很多。
